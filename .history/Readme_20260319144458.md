# 101围棋助手 (101Weiqi Helper)

面向 101 围棋网用户的浏览器扩展，提供更稳定的判题、做题统计、错题管理与棋书练习能力。

适用站点：
- https://www.101weiqi.cn/
- https://www.101weiqi.com/

## 功能概览

- 精准判题：基于页面数据状态判定对错，减少切题残留导致的误判。
- 三种模式：浏览模式、做题模式、棋书练习模式。
- 限时做题：可自定义每题限时，超时自动计错。
- 做题统计：实时显示总题数、正确/错误、超时错误、正确率。
- 错题本：本地保存每题答题历史（IndexedDB），支持待复习 / 全部 / 已刷回三种视图、按难度分组折叠展示、查看汇总统计、清空记录。
- 错题重刷：可从错题本一键进入对应题目进行重刷；在做题模式下把错题重新做对后，会自动从“待复习”列表中移除。
- 棋书搜索：支持按书名/作者/难度关键词搜索并快速跳转。
- 棋书练习：支持章节内上下题、仅错题筛选、章节进度保存与重置。

## 安装方式（开发者模式）

1. 下载或克隆本仓库。
2. 打开 Chrome/Edge 扩展管理页面：`chrome://extensions/`。
3. 开启“开发者模式”。
4. 点击“加载已解压的扩展程序”，选择项目目录 `101weiqi-helper`。
5. 打开 101 围棋题目页面，右侧会显示“101围棋助手”悬浮面板。

## 快速上手

1. 进入任意题目页。
2. 在面板中选择模式：
   - 浏览模式：仅显示状态，不计入做题统计。
   - 做题模式：启用计时与统计，记录错题。
   - 棋书练习：在棋书题目页启用章节练习能力。
3. 根据需要设置“限时(秒)”。
4. 在错题本中可使用：
  - 待复习 / 全部 / 已刷回筛选
  - 按难度分组查看
  - 一键“去重刷”跳转到原题
5. 在棋书模式下可使用：
   - 上一题 / 下一题
   - 仅错题筛选
   - 重置本章进度

## 数据存储说明

插件所有数据默认保存在本地浏览器，不上传服务器：

- `IndexedDB`：错题本与历史记录
  - DB：`101WeiqiHelperDB`
  - Store：`error_book`
  - 字段包含答题次数、待复习状态、最近结果、最近复习时间等
- `localStorage`：模式、限时、棋书缓存与章节进度
  - `weiqi_helper_mode`
  - `weiqi_helper_time_limit_sec`
  - `weiqi_helper_book_cache`
  - `book_chapter_qs:{bookId}:{chapterId}`
  - `book_progress:{bookId}:{chapterId}`

## 常见问题

### 1) 面板不显示

- 确认扩展已启用并刷新页面。
- 确认当前页面域名为 `101weiqi.cn` 或 `101weiqi.com`。
- 如果仍不显示，打开开发者工具查看是否有脚本报错。

### 2) 棋书页没显示书名/章节名

- 题目页会优先读取页面数据，读取不到时自动从面包屑提取。
- 若偶发显示为空，刷新页面后通常可恢复。

### 3) 如何清空记录

- 面板中的“清空错题本”会清空 IndexedDB 错题记录。
- 若需彻底重置，可在浏览器开发者工具中手动清除本地存储。

### 4) 错题什么时候会从待复习里消失

- 一道题在做题模式下做错后，会进入“待复习”列表。
- 如果从错题本进入重刷，并且在 `做题模式` 下重新做对，这道题会自动从“待复习”列表中移除。
- 该题不会丢失历史记录，只是从默认待复习视图切换到“已刷回”。

## 项目结构

- `manifest.json`：扩展清单（MV3）
- `content.js`：面板 UI、模式管理、错题本与棋书练习主逻辑
- `inject.js`：注入页面上下文，读取题目状态并回传
- `style.css`：悬浮面板样式

#### 为什么最终不需要 DataTable / iframe / inject.js

- 数据首次加载后完全本地化缓存，24h 内搜索零网络延迟。

### 6. 防抖机制 (Debounce)
由于 `Alpine.effect` 会在数据变化的瞬间高频触发（例如落子时，步数和结果可能在几毫秒内连续变化两次），我们引入了 50ms 的防抖函数 (`debounce`)。这确保了在一次完整的交互动作（如落子并得到结果）完成后，只向 `content.js` 发送一次判定消息，避免了性能浪费和逻辑冲突。

**新Bug记录**：如果在浏览模式下，正确情况下切到“试下”，再切回“对战”，系统会认为产生变化并错误地解开 pending 锁，导致误判为正确。
*解决方案*：在出 pending 锁读取 `duizhanResult` 时，补充条件：如果 `simulatorDuizhanPts.length === 0`，则强制判定为错（返回 2）。因为如果步数为 0，说明用户根本没有在当前棋盘上进行有效落子，此时的 `duizhanResult` 变化是由于模式切换引起的虚假变化。

**Bug：切题后新题被误判为上一题结果**：上一题做错（`duizhanResult=2`）后，切题时 Alpine.effect 被 store settle 触发，无条件解锁 PENDING，随后 scan 读到残留的 `duizhanResult=2`，新题直接误判为错。同样地，`taskinfo.result` 的破壁逻辑也可能读到上一题残留的非 0 值强行解锁。
*修复方案*：切题时记录 `_pendingSetTime = Date.now()`；effect 回调和 taskinfo 破壁逻辑均在 800ms 保护期内跳过处理。详见架构说明第 3、4 节。

**Bug：题目页面板显示「棋书 / 章节」占位符**：题目页（`/book/{bid}/{cid}/{qid}/`）无 `window.bookdata` 和 `window.nodedata`，书名/章节名无法提取。
*修复方案*：在 `inject.js` 中追加 DOM 兜底：查找面包屑中匹配 href 的 `<a>` 标签提取书名和章节名。

棋书搜索功能：那现在就只用datatable来试试，写一个功能，在原有的外观上加一个搜索框，这个用DataTable来搜索书籍，搜索结果显示书名，URL这些基本信息，点一下可跳转对应棋书页面。该搜索结果在任意页面均准确无误，可不可以做到？

**棋书搜索失败的4次尝试**（已记录，引以为戒）：
1. `fetch()` 原始 HTML → 0 条结果（表格由 DataTable + AngularJS 动态渲染，静态 HTML 不含数据行）
2. 隐藏 iframe + `iframe.contentWindow.jQuery` → 超时（内容脚本与 iframe 页面全局隔离，无法访问 iframe 的 jQuery/DataTable）
3. 向 iframe 注入内联 `<script>` → CSP 阻止（`script-src` 策略不含 `unsafe-inline`）
4. `datatable_bridge.js` 外部脚本 + postMessage → 仍超时（`event.source` 匹配问题，与 inject.js 注入时机冲突）

**最终突破**：直接 `fetch` 棋书列表页 HTML，用正则提取其中服务端**直接内嵌**在 HTML 里的 `var g_books = [...]` 变量，无需执行任何页面 JS。全部 367 本棋书数据一次性获得，本地缓存后零延迟搜索。详见架构说明第 5 节。

**Bug 修复（book 分支）：仅错题模式无法过滤出任何错题**
根本原因：`getNextBookQid` 里过滤 `doneMap` 时用的是 `String(q.qid)`，而 `recordBookResult` 写入 key 用的是 `currentProblemId`（即 `publicid`），两个字段不同，导致永远匹配不上。
修复方案：新增 `getBookQKey(q)` 辅助函数，统一使用 `publicid` 优先、`qid` 兜底的 key，`getNextBookQid` / `getPrevBookQid` 均改用此函数。

**Bug 修复（book 分支）：`getPrevBookQid` 不支持仅错题模式**
原实现在仅错题模式下仍从全序列取前一题。修复后，`getPrevBookQid` 与 `getNextBookQid` 对称支持 `bookWrongOnly`，在错题列表内向前循环。

**错题本升级：支持待复习队列与一键重刷**
错题记录结构已从单纯的“历史计数”升级为“复习状态 + 历史计数”双轨模型。新增字段包括 `needReview`、`lastResult`、`lastMode`、`lastReviewAt`，用于表示当前题目是否仍需复习、最近一次结果以及最近一次复习动作发生的时间。

*   当一道题在做题模式下做错时，会自动进入 `needReview=true` 的待复习状态。
*   当这道题通过错题本入口进入重刷，并在 `practice` 模式下做对后，会自动变为 `needReview=false`，从默认的“待复习”视图中移除。
*   错题本面板支持 `待复习 / 全部 / 已刷回` 三种筛选，并按难度分组显示，每组支持折叠，便于快速定位重点复习区间。

**错题本 UI 升级：从平铺列表改为摘要 + 分组卡片**
原先错题本是简单的单行列表，可读性较差。现已升级为包含顶部摘要卡（待复习 / 错题总数 / 已刷回）、筛选工具条、难度分组标题、单题卡片和“去重刷”按钮的结构，面板信息层级更清晰，也更适合后续扩展复习流功能。
